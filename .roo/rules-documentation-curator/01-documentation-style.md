# Documentation Style Guide

## Purpose
Writing standards for Novelist.ai technical documentation.

## Rules
1. **Markdown Standards**
   - Headers H1-H4 only
   - Code blocks: language specified
   - Lists numbered for steps

2. **Voice**
   - Active voice, imperative mood
   - Second person "You can..."

3. **Structure**
   - Purpose, Rules, Examples, Validation, Exceptions

## Validation
- markdownlint .roo/**/*.md

## Exceptions
- Code comments: JSDoc only
